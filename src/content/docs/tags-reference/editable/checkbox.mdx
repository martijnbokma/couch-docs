Editable region of `checkbox` type is used to allow the user to choose multiple options from several presented to them in the form of check boxes.

An editable region of `checkbox` type can be defined this way:

```php title="Basic Checkbox Example"
<cms:editable
    name="my_categories"
    label="Categories" desc="Check all applicable"
    opt_values='Entertainment | Computers | Sports | Health'
    type='checkbox'
/>
```

The code above will result in:

![Checkbox Example](../../../../assets/img/contents/editable-checkbox-1.gif)

The options selected in this region can be accessed, as with all other editable regions, by using the variable named after it.
Unlike its counterparts, [**dropdown**](../dropdown/) and [**radio**](../radio/) types, `checkbox` type can contain multiple selected options. In the example given above, if the user selected _Entertainment_, _Sports_ and _Health_, the following code:

```php title="Displaying Selected Values"
<cms:show my_categories />
```

will output:

```txt title="Output"
Entertainment|Sports|Health
```

As you can see, the value consists of all the three selected options clubbed together with a `|` (pipe) character separating them.

:::tip
To access each of the individual selected option, you can use the [**each**](../../each/) tag:

```php title="Iterating Through Selected Values"
<cms:each my_categories as='cat'>
    <cms:show cat /><br>
</cms:each>
```
:::

## Parameters

In addition to the parameters common to all the types of editable regions, `checkbox` accepts the following parameters:

- `opt_values`
- `opt_selected`
- `separator`
- `val_separator`

### `opt_values`

This parameter takes a list of all the options that are displayed as check boxes:

```php title="Setting Checkbox Options"
<cms:editable name="my_categories" label="Categories" desc="Check all applicable"
    opt_values='Entertainment | Computers | Sports | Health'
    type='checkbox'
/>
```

The selected options will get saved as the value of the editable region. For example, if the user selected _Sports_, the following code:

```php title="Displaying Single Selection"
<cms:show my_categories />
```

will output _Sports_.

As can be seen, the saved value is the same as the option's name. However we can specify a different value that can get saved into the editable region by using the following syntax:

```php title="Setting Custom Values"
opt_values='Entertainment=0 | Computers=1 | Sports=2 | Health=3'
```

Printing out the value contained within the editable region:

```php title="Displaying Custom Value"
<cms:show my_categories />
```

will now output `2`.

:::note
Multiple selections have been discussed above.
:::

:::tip
Specifying an empty option makes Couch generate a break `<br>`:

```php title="Adding Line Breaks"
opt_values='Entertainment=0 | | Computers=1 | | Sports=2 | | Health=3'
```

In the code above, actually seven options have been specified - three are empty (nothing between the pipes). This results in:

![Checkbox Options Example](../../../../assets/img/contents/editable-checkbox-2.gif)
:::

### `opt_selected`

You can show any option as selected by setting `opt_selected` to the value of that option:

```php title="Setting Default Selection"
<cms:editable name="my_categories" label="Categories" desc="Check all applicable"
    opt_values='Entertainment | Computers | Sports | Health'
    opt_selected='Sports'
    type='checkbox'
/>
```

Or with custom values:

```php title="Setting Default Selection with Custom Values"
<cms:editable name="my_categories" label="Categories" desc="Check all applicable"
    opt_values='Entertainment=0 | Computers=1 | Sports=2 | Health=3'
    opt_selected='2'
    type='checkbox'
/>
```

will show _Sports_ as the option selected by default.

Multiple options can be shown as selected by separating the values by a `|` (pipe character):

```php title="Setting Multiple Default Selections"
opt_selected='0 | 2'
```

The above will show _Entertainment_ and _Sports_ selected by default.

### `separator`

As seen in the examples above, the list provided to `opt_values` parameter consists of the options separated by a `|` (pipe) character and so are the values provided to `opt_selected`.
This is the default separator used by Couch. If you do not wish to have a pipe as separator (e.g. if any of the options contains the pipe character within itself), any other character can be designated as the separator by setting this parameter:

```php title="Using Custom Separator"
<cms:editable name="my_categories" label="Categories" desc="Check all applicable"
    opt_values='Entertainment=0 * Computers=1 * Sports=2 * Health=3'
    opt_selected='1 * 3'
    separator='*'
    type='checkbox'
/>
```

Note how the `opt_values` and `opt_selected` are separated by `*`.

### `val_separator`

Similar to the problem outlined above, sometimes the options in the list contain the `=` (equals to) character. This makes it impossible to use `=` between the option's name and value. You can set any other character for this purpose by setting this parameter:

```php title="Using Custom Value Separator"
val_separator=':'
```
